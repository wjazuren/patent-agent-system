# 专利交底书多智能体撰写与预审系统
## Patent Docket Multi-Agent System

---

## 📋 项目简介
本项目是一套面向科技企业研发人员、专利代理人、知识产权管理人员的**智能专利交底书生成与预审系统**。基于多智能体（Multi-Agent）协同框架，模拟真实专利代理团队的分工协作流程，自动完成技术方案解析、现有技术检索、交底书撰写、合规校验、质量评审全流程，将数天的人工撰写周期压缩至分钟级，同时大幅降低专利撰写的专业门槛。

系统采用完整的前后端分离架构：后端基于 LangGraph 构建多Agent状态机调度引擎，实现「生成-校验-评审-修正」的质量闭环；前端基于 Vue 3 + TypeScript 打造专业B端交互界面，支持生成进度实时展示、文档在线预览、历史管理与数据统计等完整功能。

## ✨ 核心特性
- **专业多智能体分工**：拆解为需求对接、现有技术检索、交底书撰写、合规校验、质量评审、文档输出6个专属Agent，完全匹配真实专利团队岗位分工
- **自迭代反馈闭环**：独立质量评审Agent终审把关，支持打回对应Agent修改，形成质量自优化闭环，最高支持3轮迭代
- **智能新颖性评估**：基于向量数据库的语义检索，自动识别对比专利，输出低/中/高三档风险等级与规避建议
- **全链路合规校验**：规则引擎 + 大模型双重校验，覆盖格式规范、公开充分性、权利要求规范性三大维度
- **可视化生成过程**：前端实时展示每个Agent的执行状态与流程日志，直观呈现多Agent协同工作流
- **生产级工程设计**：全链路结构化输出约束、统一异常降级、分层缓存、持久化存储，符合企业级应用标准
- **完整功能闭环**：配套文档管理、在线预览、一键导出复制、数据统计看板等全流程功能

## 🏗️ 系统架构
系统采用经典五层分层架构，各层职责解耦，支持横向扩展新的Agent角色与能力组件。

```
┌─────────────────────────────────────────────────────┐
│  交互层：Vue 3 前端页面 / FastAPI RESTful 接口        │
│  接收用户请求，承接对话式修改，返回最终交付物          │
├─────────────────────────────────────────────────────┤
│  调度层：LangGraph 全局状态机调度器                    │
│  任务拆解、节点分发、流程管控、条件分支、闭环回退      │
├─────────────────────────────────────────────────────┤
│  专业Agent层：6个领域专属智能体                        │
│  需求对接 / 现有技术检索 / 交底书撰写                │
│  合规校验 / 质量评审 / 文档输出                      │
├─────────────────────────────────────────────────────┤
│  工具能力层：公共能力组件                              │
│  LLM调用工具 / RAG检索工具 / 专利检索工具             │
│  缓存工具 / 格式校验工具 / Token统计工具              │
├─────────────────────────────────────────────────────┤
│  数据存储层：多介质存储体系                            │
│  Chroma 向量库 / Redis 缓存 / SQLite 持久化          │
└─────────────────────────────────────────────────────┘
```

### 智能体角色分工
| Agent角色 | 对应真实岗位 | 核心职责 |
|:---|:---|:---|
| 需求对接Agent | 专利客户经理 | 解析口语化技术描述，结构化提取创新点、技术问题与核心方案 |
| 现有技术检索Agent | 专利检索员 | 多源语义检索现有技术，评估新颖性风险，输出对比文件清单 |
| 交底书撰写Agent | 专利代理人 | 基于模板与检索结果，分模块生成标准规范的交底书全文 |
| 合规校验Agent | 流程审查员 | 格式规范、法条合规、公开充分性三重校验，输出修改建议 |
| 质量评审Agent | 质检经理 | 综合终审，决定通过或打回对应Agent修改，把控最终质量 |
| 文档输出Agent | 文档专员 | 生成标准化Markdown交付文档，归档入库并记录统计数据 |

## 🛠️ 技术栈详解
### 后端技术栈
| 技术领域 | 选型方案 | 选型说明 |
|:---|:---|:---|
| 多Agent调度 | **LangGraph** | 业界标准状态机调度框架，原生支持条件分支、回退边、并行流程，比线性链式调用更专业 |
| 大模型适配 | LangChain + OpenAI 兼容接口 | 无缝适配通义千问、文心一言等主流中文大模型，国内直连稳定 |
| Web服务框架 | **FastAPI** | 高性能异步RESTful框架，自动生成API文档，原生支持Pydantic数据校验 |
| 结构化约束 | **Pydantic v2** | 全链路Agent输出格式强约束，从根源避免大模型自由文本导致的系统不稳定 |
| 开发语言 | Python 3.10+ | AI领域生态最完善，适配所有大模型、向量库、Agent框架工具链 |

### 前端技术栈
| 技术领域 | 选型方案 | 选型说明 |
|:---|:---|:---|
| 核心框架 | **Vue 3 + Vite** | 现代前端技术组合，开发环境秒启动，热更新速度快，开发体验优异 |
| 类型系统 | **TypeScript 5** | 全链路类型安全，接口响应、组件Props、状态管理全覆盖，减少运行时错误 |
| UI组件库 | **Element Plus** | 国内企业级B端首选组件库，组件丰富，原生TS支持，风格专业统一 |
| 状态管理 | **Pinia** | Vue官方推荐新一代状态管理，API简洁，类型推断友好，比Vuex更易维护 |
| 路由管理 | Vue Router 4 | 官方路由方案，支持动态路由、路由守卫，适配单页应用需求 |
| 文档渲染 | markdown-it + highlight.js | 轻量可扩展，支持代码块高亮，完美渲染专利交底书Markdown内容 |
| 数据可视化 | **ECharts 5** | 功能完善的开源图表库，类型支持完善，满足统计看板各类图表需求 |
| 样式方案 | SCSS + CSS变量 | 支持变量、嵌套、主题定制，样式可维护性强 |

### 数据存储与中间件
| 技术领域 | 选型方案 | 选型说明 |
|:---|:---|:---|
| 向量数据库 | **Chroma** | 轻量嵌入式向量库，支持语义检索，无需独立部署服务，项目开箱即可运行 |
| 缓存中间件 | **Redis** | 分层缓存高频检索结果、模板数据，大幅提升响应速度，降低大模型调用成本 |
| 持久化存储 | **SQLite** | 轻量文件型关系数据库，存储历史文档、系统统计数据，无需额外部署数据库服务 |

## 🚀 快速开始
### 环境要求
- Python 3.10 及以上版本
- Node.js 18 及以上版本
- Redis 6.0+（可选，用于缓存加速，不开启也可正常运行）

### 一、后端服务部署
1. **进入后端目录**
```bash
cd patent_agent_system
```

2. **创建虚拟环境并安装依赖**
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows 系统:
venv\Scripts\activate
# Linux / Mac 系统:
source venv/bin/activate

# 安装全部依赖
pip install -r requirements.txt
```

3. **配置环境变量**
```bash
# 复制环境变量模板
cp .env.example .env
```
编辑 `.env` 文件，填入你的大模型 API 密钥等配置：
```env
# 大模型配置（以通义千问为例）
LLM_API_KEY=你的大模型API密钥
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen3.6-plus

# 其他配置可保持默认
```

4. **初始化知识库（首次运行执行一次即可）**
```bash
python init_knowledge_base.py
```

5. **启动后端服务**
```bash
uvicorn app.api.main:app --host 0.0.0.0 --port 8000 --reload
```
服务启动后，访问 `http://localhost:8000/docs` 可查看自动生成的 Swagger API 文档，在线调试所有接口。

### 二、前端页面部署
1. **进入前端目录**
```bash
cd ../patent-agent-frontend
```

2. **安装项目依赖**
```bash
npm install
```

3. **启动开发服务器**
```bash
npm run dev
```
启动成功后，访问 `http://localhost:5173` 即可进入系统前端页面。

## 📡 核心API接口
| 接口路径 | 请求方法 | 功能描述 |
|:---|:---|:---|
| `/patent/generate` | POST | 提交技术方案描述，启动多Agent流程生成交底书 |
| `/patent/{document_id}` | GET | 根据文档ID查询完整交底书详情 |
| `/patent/list` | GET | 分页获取历史生成的文档列表 |
| `/stats` | GET | 获取系统运行统计数据 |
| `/health` | GET | 系统健康状态检查 |

## 🖥️ 功能页面说明
1. **生成交底书页**：技术方案文本输入、快捷示例填充、多Agent执行进度实时可视化、生成结果摘要展示
2. **文档管理页**：历史文档卡片式列表、关键词搜索、风险等级筛选、分页浏览
3. **文档详情页**：Markdown格式交底书在线渲染预览、一键复制全文、导出Markdown文件、Token消耗明细
4. **数据统计页**：累计生成量、Token总消耗、风险等级分布、近7日生成趋势等多维度数据看板
